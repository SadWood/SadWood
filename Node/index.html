# 1.2、node 服务器搭建

node.js服务器构建步骤

1、加载内置 http 模块

2、构建服务器请求主体

​ 2.1、编写服务器构建代码

3、启动并监听服务器端口

// 1、加载内置 http 模块
const http = require("http");

// 2、构建服务器请求主体
// 爬虫:
const server = http.createServer(function(request, response) {
  // request 用于解析用户请求的
  console.log(request.url);
  // 	2.1、编写服务器构建代码
  // 需要定义用户请求到达后，服务器需要完成的事情
  console.log("用户访问了服务器");
  // 服务的构建，需要注意 用户的每次访问 都需要等待服务的响应结果
  // 响应头的设置
  response.writeHead(200, {
    "Content-Type": "text/html; charset=utf-8"
  });
  // write() 设置响应页面
  response.write("<h1>node服务器运行</h1>");
  // 告知请求 访问结束
  response.end();
});

// 3、启动并监听服务器端口(定义服务器端口)
//    node 服务器的构建 默认端口 应该使用 3000  ==>  8080  ==> 8000
//    tomcart（java web） 服务器的默认端口 8080
//    PHP 服务器默认端口  80
//    mysql 服务器的端口  3306
server.listen(8080, function() {
  console.log("服务器启动并监听端口");
});

  • 堆栈概念
// 1、http 的加载

// 问题2：开发时是否可以随便定义一个字符，然后得到一个对象

  • http 字符串在 node 环境中 就是一个 内置的模块
  • 什么是模块：是一个可以被node 运行的 代码的集合==> 指通过js 方式编写的代码
    • 限制在文件：以 .js .json .node 结尾的文件 都可以作为 nodejs 的模块使用
  • 对于遵循JS语法定义的 js 文件，只要是运行在 node环境中 就可以被称之为模块

# 2.2、模块导出

# 2.2.1、exports 的导出

  • exports 属性 是nodeJS 为所有js文件提供一个全局的属性定义
    • 全局的属性 调用方式 类似于全局方式，但是 使用时 指向的是 当前文件的作用域范围
  • exports 是一个 只读属性 => 不可被重写 ==> 一旦被重写，此时exports 将由全局变量转为 局部变量
  • **exports 是一个已经被定义的对象 **
exports.属性名 = 属性值

# 2.2.2、module.exports 的导出

  • module 是 exports 属性的 父级对象
  • module.exports 实际上就是 模块中的 exports 对象
  • module.exports 可以完成 exports 对象的重写
  • 扩展node模块的 主要扩展功能
  • 如果模块定义过程中，使用了module.exports 的方式，会造成如下影响
    • 1、使用的是 module.exports.属性名 = 属性值 ,对返回的 对象结果 进行 属性增加
    • 2、使用的是 module.exports=值 ， 导致 当前模块中 所有 使用 exports 导出的属性全部丢失
var require = function(path) {
  // 根据 path 指定的路径进行文件加载，对文件中的代码进行执行
  //  node 的内置组件 fs  ==>  读取文件

  // 使用下面的定义方式 完成 对模块产生需要使用的变量
  let module = {
    exports: {}
  };

  (function(module, exports) {
    // 文件加载后 倒入的 文件代码
    exports.name = "tom";
    exports.age = 23;
    exports.arr = [1, 2, 3, 4, 5, 6];
    exports.obj = { a: "aa", bb: "bb" };
    exports.fun = function() {};

    module.exports = {
      ss: "sss"
    };
    // 文件加载后 倒入的 文件代码
  })(module, module.exports);

  return module.exports;
};

# 2.2.3、node 模块包装器

  • 包装器主要完成 对 当前模块文件运行过程中，需要的 变量进行初始化
  • 对模块中的 变量 进行作用域的限制

(function(exports, require, module, __filename, __dirname) {
  var arr = [1, 2, 3, 4, 5, 6];

  for (let i = 0; i < arr.length; i++) {
    const item = arr[i];
    console.log(item);
  }
})(exports, require, module, __filename, __dirname);

# 3、node的内置调试工具

  • 可以完成 将 js 代码逐行执行
  • 通过 node debug 文件名 ==> 启动调试工具
    • cont, c - 继续执行（将代码完全执行完成）。
    • next, n - 下一步 （将代码一行一行执行）。
    • step, s - 跳进函数。
    • out, o - 跳出函数。
    • pause - 暂停运行代码（类似开发者工具中的暂停按钮）。
