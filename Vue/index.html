# 1、设计模式

# 1.1、SPA (设计理念，不是技术)

  • 单页应用程序（single page web application）
    • 对于用户视角：浏览器没有进行刷新的同时，可以访问多种 页面内容
      • 用户一开始 访问了一个主页，当主页加载完毕后，用户去访问当前主页中任意链接，都不再会出现浏览页面刷新或跳转的状态
    • 程序员的角度：可以被用户直接访问的页面只有一个（主体页面），其他内容以模板的形式在该页面中进行书写，通过JS对于文件中DOM的解析和写入方式实现内容的切换
    • 程序运行过程中，主体装载页面（用户的访问页），只有一个，页面中的实际内容用模板文件进行定义，并通过技术手段完成JS DOM的动态渲染

# 1.2、MVVM （框架模型）

  • Model–view–viewmodel模型视图视图模型
  • 提供一种开发思路，降低 开发者在程序编写过程中，对于 DOM的 操作依赖==>将所有的DOM操作进行封装，开发者在完成页面中数据的动态渲染时 ，无需关心DOM操作

  • 将程序员不用编写DOM操作代码 ，直接定义数据和模板 ，即可完成页面和数据的交互操作

# 2、vue的简介

  • Vue 就是一个实现了MVVM设计思想和完成SPA设计概念的一个前端框架

# 3.3、指令

  • Vue的指令，是以 标签属性的 方式 定义在 标签上的，具有 特殊处理能力的 功能方法
  • 指令的底层是 处理方法
  • 指令在 页面中 以 属性的方式定义在 标签上
  • Vue的指令 以 v- 开头进行指令属性描述，v-指令名称（函数名），为定义了指令的 DOM元素增加处理功能，指令实际上就是 Vue Html 编译触发器
  • 指令存在 boolean 类型指令 ==> 写表示 true 不写表示 false

指令使用语法

  • 基本语法：<p v-指令名="变量名|表达式…… （插值表达式的所有使用方式） " ></p>
  • 指令参数：<p v-指令名：参数="指令取值"></p>，在指令名称后，以 ： 表示的为 指令参数
    • 指令参数 用于描述指令运行依赖
    • 指令参数只能定义一个
  • 指令修饰符：<p v-指令名.修饰符="指令取值"></p>，在指令名称后，以 . 方式表示的参数
    • 用于描述指令的运行功能
    • 可以有多个，修饰符的执行顺序和 定义顺序有关
  • 完整语法<p v-指令名:指令参数.修饰符1.修饰符2……="指令取值" ></p>:

# 3.3.4、v-on

  • 事件绑定：完成是 指定 vue 对象独有方法的 事件绑定

    • 1、依赖指令参数 进行 触发事件名称的定义
    • 2、v-on 完成事件绑定，使用的技术是 动态事件绑定
    • 3、v-on 完成的事件绑定，绑定的 方法的 参数 可以直接使用 vue实例中的数据变量
    • 4、v-on 完成的事件绑定，无法在HTML标签中 通过 this 传参的方式获取当前DOM对象 （this==>window）
    • 5、v-on 完成的事件绑定，将无法在方法参数中 直接使用 event 进行事件源的传递，需要使用 $event 进行传递

  • v-on 修饰符：

    功能修饰符

    • .stop - 调用 event.stopPropagation()。 阻止冒泡
    • .prevent - 调用 event.preventDefault()。阻止默认事件
    • .capture - 添加事件侦听器时使用 capture 模式。改变冒泡事件的执行顺序
    • .self - 只当事件是从侦听器绑定的元素本身触发时才触发回调。

    操作修饰符

    • .once - 只触发一次回调。一次性事件定义 ==> 事件是 完成事件触发后的 事件解绑

    过滤修饰符

    • .{keyCode | keyAlias} - 只当事件是从特定键触发时才触发回调。
      • keyCode 叫做键盘码 a ：65 f1 : 112
      • keyAlias 键盘映射表 ==> vue 内置
        • v1.0 存在键盘映射表
        • v2.0 键盘映射表被删除
        • v2.3 键盘映射表被加回
    • .left - (2.2.0) 只当点击鼠标左键时触发 | 方向键的左键
    • .right - (2.2.0) 只当点击鼠标右键时触发。 | 方向键的右键
    • .up - 方向键的上键
    • .down - 方向键的下键
    • .middle - (2.2.0) 只当点击鼠标中键时触发。

    组件修饰符

    • .native - 监听组件根元素的原生事件。
    • .passive - (2.3.0) 以 { passive: true } 模式添加侦听器

  • Vue 对象中的 methods 属性的特点

    • 在methods 中定义的方法 ，默认 this 指向的是 当前Vue 的对象
    • 可以在方法中 通过 this 去访问 当前对象中的 数据 或者 函数

# 4、计算属性

  • computed 和 data 具有相同 数据仓库 功能 的同时 也具有 methods 的操作功能
    • data 存放的是 初始数据值
    • computed 存放的是 值得 计算过程 ==> Function
  • 作用，在页面获取计算属性中的变量时，获取是 运行后的返回值
    • 定义时，需要注意return 在页面中将被展示的数据
  • 计算属性在单独使用 ，将不具有任何意义，需要对其他数据 （data 中的变量） 进行配合处理的
    • 计算属性的使用 依赖于 其它的基础变量
  • 为基础数据提供判断的区域
  • 计算属性 默认是 只读的 默认只提供 get 方法 （默认不能被 v-model使用）
  • 计算属性如果需要 实现 读写操作，在定义时 需为当前属性 提供 两个特殊方法 set 和 get
  • 为了组件开发提供处理接口（get 和 set）（后续课程）

# 8.3、过滤器 vs 计算属性

过滤器 计算属性
不依赖于任何数据 要让计算属性具有意义，一定需要依赖一个变量
可以定义全局和局部两种，用于工具定义或者 功能 只能在局部完成 计算后属性的结果输出
过滤器可以接收额外参数 不能接收额外参数（可以依赖多个变量）
过滤器方法只要在页面被定义，就会调用执行方法，得到的结果不会进行缓存处理 计算属性在页面被定义时，会判断是否第一次调用，如果是执行方法，如果不是 会做缓存获取； 在后续使用中，只有 依赖变量发生变换时，才会重新调用方法并完成缓存替换

  // Vue.directive( id, [definition] )
  //  id:描述定义的指令名
  //  definition：描述指令功能
  //    object 方式定义参数，表示将拆分指令的生命周期进行单独定义
  /*
    指令的生命周期，指的是 指令从定义到执行 到 销毁的过程
    {
      // key：描述指令的生命周期名
      // value: 执行函数
      // 指令第一次被绑定到页面元素时 所调用的方法 ==> 只会被执行一次
      bind: function () {},
      // 被绑定的元素插入到父节点中时调用的指令
      inserted: function () {},
      // 所绑定VNode 发生更新时 会调用的方法
      update: function () {},
      // 组件Vnode发生变化时 会调用的方法  ？ ==> 组件开发
      componentUpdated: function () {},
      // 指令被解绑(销毁)时所调用的函数
      unbind: function () {}
    }
    对于指令的生命周期而言，所有的 钩子方法都可以接受到 四个参数
    el,binding,vnode,oldvnode
      el：指向于 调用该指令的 DOM 元素
      binding: 存放指令使用时的 参数 修饰符 取值
      vnode: 描述更新后 的新 虚拟DOM
      oldVnode：描述 更新前的 旧 虚拟DOM
  */
  // function  ==> 简写方式  替代 bind update

  Vue.directive("test", {
    bind(el, binding, vnode, oldvnode) {
      console.group("==>bind")
      console.log(el)
      /*
        arg:"aaa"  ==> 指令的参数
        def:{bind: ƒ, inserted: ƒ, update: ƒ, unbind: ƒ}
        expression:" 'ssss' "  ==> 指令所获取的数据的表达式
        value:"ssss"  ==> 指令所获取的数据的表达式的执行结果
        modifiers:{d:true,e:true,f:true }  ==> 表示指令启用的修饰符
        name:"test"
        rawName:"v-test:aaa.d.f.e"
      */
      console.log(binding)
      console.log(vnode)
      console.log(oldvnode)
      console.groupEnd();
    },
    inserted(el, binding, vnode, oldvnode) {
      console.group("==>inserted")
      console.log(el)
      console.log(binding)
      console.log(vnode)
      console.log(oldvnode)
      console.groupEnd();
    },
    update(el, binding, vnode, oldvnode) {
      console.group("==>update")
      console.log(el.style.color = "red")
      console.log(binding)
      console.log(vnode, vnode.elm.innerHTML)
      console.log(oldvnode, oldvnode.elm.innerHTML)
      console.groupEnd();

    unbind(el, binding, vnode, oldvnode) {
      console.group("==>unbind")
      console.log(el)
      console.log(binding)
      console.log(vnode)
      console.log(oldvnode)
      console.groupEnd();
    }
  })
  Vue.directive("aa", function (el, binding, vnode, oldvnode) {
    console.log("指令调用")
  });
