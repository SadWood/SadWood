# 一、基础命令

  1. master ：默认开发分支 . origin ：默认远程版本库 . Head ：指向默认开发分支 Head===>master . Head^ ：Head的父提交

# 1.1、创建仓库

命令 作用
git clone <url> 克隆远程版本库
git init 初始化本地版本库

# 1.2、修改&提交

命令 作用
git status 查看状态
git diff 查看变更内容
git add . 跟踪所有改动过的文件
git add <file> 跟踪指定文件
git mv <oldname> <newname> 文件重命名
git rm <file> 删除文件
git rm --cached <file> 停止跟踪文件但不删除
git commit -m "message" 提交所有文件
git commit --amend 修改最后一次提交

# 1.3、查看提交历史

命令 作用
git log 查看提交历史
git log -p <file> 查看指定文件的提交历史
git blame <file> 以列表方式查看指定文件的提交历史

# 1.4、撤销

命令 作用
git reset --hard HEAD 撤销工作目录中所有未提交文件的修改内容
git checkout HEAD <file> 撤销指定的未提交文件的修改内容
git revert <commit> 撤销指定提交

# 1.5、分支与标签

命令 作用
git branch 显示所有本地分支
git checkout 切换到指定分支或标签
git branch <new-branch> 创建新分支
git branch -d <branch> 删除本地分支
git tag 列出所有本地标签
git tag <tagname> 基于最新提交创建标签
git tag -d <tagname> 删除标签

# 1.6、合并与衍合

命令 作用
git merge <branch> 合并指定分支到当前分支
git rebase <branch> 衍合指定分支到当前分支

# 1.7、远程操作

命令 作用
git remote -v 查看远程版本库信息
git remote show <remote> 查看指定远程版本库信息
git remote add <remote> <url> 添加远程版本库
git fetch <remote> 从远程库获取代码
git pull <remote> <branch> 下载代码及快速合并
git push <remote> <branch> 上传代码及快速合并
git push <remote> :<branch/tag-name> 删除远程分支或标签
git push --tags 上传所有标签

# 二、组合命令

# 2.1、删除远程已忽略的文件

  1. 把需要删除的文件加入.gitignore . git rm -r --cached xxx.xxx 停止跟踪文件但不删除 . git commit -m "ignore xxx.xxx” 重新提交 . git push 重新上传合并

# 2.2、解决git中文显示的问题

git config --global core.quotepath false

# 2.3、换行符错误提示

不同操作系统下，处理行尾结束符的方法是不同的：

  • Windows下：CRLF（表示句尾使用回车换行两个字符，即windows下的"\r\n"换行）

  • Lunix下：LF（表示句尾，只使用换行）

  • Mac下：CR（表示只使用回车）

git config --global core.autocrlf false

如果设置为true，在Windows下则会将CRLF转为LF
