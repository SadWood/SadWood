# 1、组件

  • 组件对于页面而言，就是代码中一组自定义标签，在项目充当模板功能，完成页面功能的重复调用以及加载 -- 为项目提供代码复用接口
  • 快速装载和使用，一个页面实际上是由多个组件构成
  • 组件的开发不再是单纯页面绘制，包含功能的实现，包含复用的定义
    • 针对页面展示：样式适配，JS 标准适配，布局适配（弹性盒子）
    • 针对于功能：模板的切换，小插件的实现（评分插件）

# 2.3.2、子组件向父组件传递数据

  • 显式数据传递 ==> 显式：开发者 自行定义的代码 隐式：工具插件、js内置功能…… 提供的 处理方式
  • 依赖于子组件标签事件的绑定，$emit的事件触发
<script src="../vue.js"></script>

<body>
  <div id="app">
    <h3>父组件</h3>
    <hr>
    <!--
        在子组件标签上绑定的时间不会像属性一样 传递给子组件的根标签
        ==> 事件绑定和属性绑定处理方式相反
        ==> 如果为子组件标签绑定的事件不被放行，认为该事件是 子组件标签（对象）的事件

        ==> 1、放行 子组件标签的 绑定事件
        ==> 2、事件不被放行 将绑定在 组件对象上
          未被放行的事件 将作为一个没有触发器的事件
          定义在子组件对象中 _events 事件仓储区 将记录组件的 所有未被放行的事件
          在_events 存放的是事件对象 而非方法
          ==> 如何触发事件对象

          vue 为 所有的实例提供一个实例方法，用于触发 组件的事件仓库仓储区 中定义的事件
          vm.$emit()

        子组件标签上绑定未被放行的事件，可以通过 两种方式将子组件数据传递给父组件
          1、v-on：事件名=“方法（$event）” 方式手动传递事件源
          2、v-on：事件名= “方法” 方式 自动传递事件源

        数据传递为一次性触发
     -->
    <type-list @click.native="show($event)" @aaa="print($event)" @bbb="print"></type-list>
  </div>
  <template id="typelist">
    <div>
      <h3>子组件</h3>
      <input type="text" v-model="info">
    </div>
  </template>
  <script>
    new Vue({
      el: "#app",
      data: {
        msg: ""
      },
      methods: {
        show(e) {
          console.log("触发事件", e)
        },
        print(e) {
          console.log(e);
          this.msg = e.info;
          console.log("print方法")
        }
      },
      components: {
        TypeList: {
          template: "#typelist",
          data() {
            return {
              info: "数据"
            }
          },
          mounted() {
            /*
              $emit(eventName, event)
              eventName: 组件接收内置事件的事件名
              event: 触发时间时 所定义的事件源对象
            */
            this.$emit("bbb", {
              info: this.info
            })
          },
        }
      }
    });
  </script>
</body>

# 2.3.3、非父子组件的数据传递

  • 依赖 共有 父组件 == > root组件
    • 父组件向子组件 使用 属性绑定方式传递数据
    • 子组件向父组件 使用 事件绑定方式传递数据

  • 层级过程的组件数据传递 （逻辑复杂切不利于维护）（不推荐使用这种方式完成数据共享）

  • 中央数据总线

    • 定义的仓库 必须是一个vue 对象
    • 该对象 一定可以被 所有的vue 对象直接获取
    • 可以满足上述要求的vue 对象只有一个 ： 根组件
      • 他是一个 vue 对象
      • vue为所有组件 装载一个 特殊的实例 属性 $root ==> 指向的是根对象
    • 依赖于两个实例方法
      • $emit 事件触发 （被定义在 需要发送数据的组件中）
      • $on 事件绑定 （被定义在 需要接收数据的组件中）

# 1、通过计算属性 打破单项数据流

<script src="../vue.js"></script>

<body>
  <div id="app">
    <h1>父组件</h1>
    <h3>{{ info }}</h3>
    <input type="text" v-model="info">
    <hr>
    <hello :info="info" @sendinfo="setInfo"></hello>
  </div>
  <template id="hello">
    <div>
      <h1>子组件</h1>
      <h3>{{ infoData }}</h3>
      <input type="text" v-model="infoData">
    </div>
  </template>
  <script>
    new Vue({
      el: "#app",
      data: {
        info: "被共享的数据"
      },
      methods: {
        setInfo(e) {
          this.info = e.info;
        }
      },
      components: {
        Hello: {
          template: "#hello",
          props: {
            info: ""
          },
          computed: {
            infoData: {
              get() {
                return this.info;
              },
              set(value) {
                console.log(value)
                this.$emit("sendinfo", {
                  info: value
                });
              }
            }
          }
        }
      }
    });
  </script>
</body>
